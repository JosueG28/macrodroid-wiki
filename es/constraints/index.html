    Constraints in MacroDroid¶

    Constraints are real-time conditions that must be true when a trigger fires for a macro to run.
    While triggers define when something could happen, constraints decide if it should — based on things like time, charging status, or location.

    → Full Constraints List

    What Is a Constraint?¶

    A constraint is a check that runs only at the moment a trigger activates.
    If all constraints pass, the macro continues. If any fail, it’s canceled.

    Key facts: - Checked only at trigger time - Uses AND logic — all must be true - Doesn’t monitor continuously

    Example:
    Trigger: Headphones Connected
    Constraint: Media Volume < 30%
    → Runs only if both are true at the same time.

    Using Multiple Constraints¶

    You can add as many constraints as needed — but remember:

    • All must be true
    • They’re evaluated simultaneously
    • You can combine types (e.g. time + battery + location)

    Example:
    Trigger: Screen On
    Constraints:
    - Time: 10:00 PM–7:00 AM
    - Charging: No
    - Wi-Fi: Connected
    → Runs only during the night, on battery, and with Wi-Fi.

    Where Constraints Apply¶

    Constraints can be attached at different levels:

    Local to Triggers¶

    Affect only a specific trigger.

    Example:
    Trigger: Screen On
    Constraint: Charging = No
    → Trigger fires only if not charging.

    Local to Actions¶

    Affect only a specific action.

    Example:
    Action: Set Brightness to 100%
    Constraint: Time = Daytime
    → Runs only during the day.

    Global Constraints¶

    Affect the entire macro — added in the Constraints section.

    Example:
    Global: Wi-Fi SSID = Home
    → Macro won’t run unless you’re on your home network.

    Summary of Constraint Scopes¶

    Type Affects Scope Example
    Trigger Local A trigger That trigger only Only run if volume > 50% when headphones connect
    Action Local An action That action only Only send SMS if battery > 20%
    Global Entire macro All parts Block macro during work hours

    Real-Life Examples¶

    Morning Alarm¶

    • Trigger: Alarm at 7:00 AM
    • Global Constraint: Day = Weekdays
    • Actions:
    • Increase volume
    • Read weather forecast
    • Turn on smart plug
      • Action Constraint: Plug is connected

    Location Reminder¶

    • Trigger: Enter grocery store (geofence)
    • Global Constraint: Time = 8:00 AM – 8:00 PM
    • Action: Show “Buy Milk” notification
    • Action Constraint: Notification not already shown

    Constraint Categories¶

    Category Example Conditions
    Device State Screen on/off, Charging, Headphones plugged
    Connectivity Wi-Fi SSID, Bluetooth device active
    Time/Location Time range, Day of week, Geofence entered
    System Battery level, App in foreground, Notification shown
    Custom Logic Variable value, Macro not running

    Why Constraints Matter¶

    • Precision – Run macros only in the right conditions
    • Efficiency – Save battery and system resources
    • Simplicity – Avoid duplicating macros
    • Control – Combine with variables for smart logic

    Triggers vs Constraints¶

    Triggers Constraints
    Detect events (e.g., SMS received) Check current states (e.g., charging)
    Start the macro Decide if it should run
    Support OR logic Use AND logic only
    React to changes Evaluate conditions once, on fire
